搭建连接器指南

对连接器的文件结构有了一个初步概念之后，让我们讨论一下开发、实现一个连接器的思路。

在连接器对接的服务商网站注册一个账号​

仔细浏览服务商的网站和相应的管理页面，总是我们开始实现一个连接器的第一步。在这里你可以找到关于连接器服务提供商的一些基本信息。

查找讲解连接器要对接的服务的功能和工作流程的文档​

由于不同类型的连接器对应着不同的使用场景，我们在搜索相关的文档的时候也需要注意用不同的关键词。

社交连接器 是用于引导终端用户授权登录的；而 短信连接器 和 邮件连接器 适用于调用发送短信或邮件服务 API 的。

提示

社交连接器 文档关键词：授权流程，获取用户档案以及 OAuth 2.0 协议（大部分 社交连接器 对接的服务商都遵从此协议）。
短信连接器 和 邮件连接器 文档关键词：集成发送短信（或邮件）API。

实现以及关键信息的组织​

constant.ts​

constant.ts 中的一些常见信息：

  • 连接器的默认元数据
  • 连接器服务提供商 API 对应的端点
  • 授权请求的 scope

types.ts​

types.ts 中的一些常见的变量：

  • 连接器配置的类型检查
  • 请求的参数类型定义
  • 响应的参数类型定义
  • 错误类型定义
备注

我们 强烈建议 使用 Zod 来建立对连接器配置、请求和响应的类型保护。

Zod 在配置连接器的时候会非常有用，如果配置中出现了类型错误，弹窗会展示从 ZodError 拿到的详细报错信息。

utils.ts^​

通常包括了一些工具函数，例如：

  • 支付宝连接器有（区别于其他连接器）特有的，给请求的参数签名的流程
  • 阿里云连接器对于响应的参数有特殊的处理流程
备注

按需创建 utils.ts 文件，并且在创建该文件的同时，建议添加相应的单元测试到 utils.test.ts（不做强制要求）。

index.ts​

连接器类型是对于 SMSConnector，EmailConnector 和 SocialConnector 接口的一个实现。

这里有对于不同类型连接器接口的详细定义。

interface BaseConnector<T> {
  metadata: ConnectorMetadata;
  getConfig: GetConnectorConfig;
  validateConfig: ValidateConfig<T>;
}

interface SmsConnector<T> extends BaseConnector<T> {
  sendMessage: SmsSendMessageFunction;
  sendTestMessage?: SmsSendTestMessageFunction;
}

interface EmailConnector<T> extends BaseConnector<T> {
  sendMessage: EmailSendMessageFunction;
  sendTestMessage?: EmailSendTestMessageFunction;
}

export interface SocialConnector<T> extends BaseConnector<T> {
  getAuthorizationUri: GetAuthorizationUri;
  getUserInfo: GetUserInfo;
}

若是要了解连接器接口的设计和连接器类型中方法的定义，参见 @logto/connector-types。

关于 ConnectorMetadata 的参考，见「连接器 - ConnectorMetadata」。

  • 连接器的 validateConfig 方法对于所有类型的连接器都是必须的。
  • 所有 邮件连接器 和 短信连接器 都需要 sendMessage 方法来调用发送信息的 API（使用的是数据库中的连接器配置）。开发者可以选择性地实现 sendTestMessage 方法，来在设置连接器时使用尚未保存的配置发送测试信息。
  • 授权 URL 生成器 getAuthorizationUri 方法以及获取用户档案的 getUserInfo 方法对大多数 社交连接器 都是必要的。
  • 所有的 社交连接器 都应该有用用户授权拿到的 authorization code（授权码）交换得到 access token（访问令牌）的 getUserInfo 方法。
提示

连接器的类型实现具体应该包含哪些方法，因其类型而异。

想知道为什么 社交连接器 一定要包含某些特定的方法？
来看看「OAuth 2.0 协议」吧！

备注

对于大多数 社交连接器 来说，在终端用户授权的前提下最终获取到用户档案的流程可以分为两个步骤（假设终端用户授权成功）：

  1. 通过连接器服务商授予的 authCode 获取 accessToken。
  2. 用 accessToken 请求可以公开访问的用户档案。

恭喜！经过了以上的步骤，你的连接器应该准备好投入生产了，去试试吧！🚀 🚀 🚀
