连接器的文件结构

下面的文件树呈现了每一个连接器作为一个 TypeScript 项目是如何组织内部文件的。

以 ^ 结尾的文件是可选的文件，剩下的都是必需的文件。

ConnectorRootDir/
├── package.json
├── tsconfig.build.json
├── tsconfig.test.json^
├── README.md
├── logo.svg
├── logo-dark.svg^
├── docs/
│   └── configTemplate.json
└── src/
    ├── index.ts
    ├── index.test.ts
    ├── types.ts
    ├── constant.ts
    ├── mock.ts^
    ├── utils.ts^
    ├── utils.test.ts^
    └── ...

基于上面呈现的连接器文件结构，让我们浏览一遍，看看每个文件都是用来干嘛的，以及它们之间是怎么共同协作的。

package.json​

package.json 包含了当前连接器仓库的各种配置。

tsconfig.*.json​

tsconfig.*.json 表示了编译需要用到的一些配置文件，包括了 tsconfig.build.json 以及 tsconfig.test.json。

tsconfig.build.json 用来定义工程「构建」时的配置，而 tsconfig.test.json 用来定义工程「测试」时的配置。但由于不是所有的工程都有「测试」流程但都有「构建」流程，所以 tsconfig.test.json 不是必须的而 tsconfig.build.json 是必须的。

提示

为了正确配置 package.json 和 tsconfig.*.json 文件，你可以参考其他已经存在的连接器实现。

README.md​

README.md 是关于如何设置连接器配置的详细教程。

logo.svg​

logo.svg 是保存连接器矢量图 logo 的文件。

logo-dark.svg​

logo-dark.svg 是保存连接器深色模式 logo 矢量图的文件。

备注

查看 logo，了解 logo.svg 和 logo-dark.svg 之间的关系。

configTemplate.json​

configTemplate.json 给出了连接器配置的一个样例。

index.ts​

index.ts 是包含了连接器类型具体实现的文件。

index.test.ts​

index.test.ts 包含了 index.ts 实现逻辑相应的单元测试。

types.ts​

types.ts 文件应包含关于变量和方法类型的定义。

constant.ts​

constant.ts 理应包含关于连接器元数据，服务商提供的 API 端点等和连接器相关的各种常量。

mock.ts​

mock.ts 文件中应包括一些用于测试目的的模拟值。

utils.ts​

utils.ts 中，如有必要，应放一些与业务相关性不大的工具函数，而 utils.test.ts 中包含工具函数相应的一些单元测试。
