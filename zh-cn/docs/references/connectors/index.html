🔌 连接器

介绍​

什么是连接器？​

连接器 在 Logto 中扮演着重要的角色。在他们的帮助下，Logto 使得终端用户能够使用无密码注册登录和社交帐号登录。在网站和应用逐渐渗透到我们的生活中的大背景下，无密码登录和社交登录让用户们能够避免为了管理各种各样的帐号而感到头大。

😯 是不是听起来非常引人入胜？ 如果你想要更改一个已有连接器的设置，请参考 配置连接器。如果找不到想要更改设置的连接器，你可以根据 创建连接器 来开发属于你的连接器。

组成​

连接器数据中有很多属性。

为了提升数据的加载和更新效率，我们把一部分经常更新的连接器数据托管到数据库，剩下的一部分保存在本地缓存中。

  • 本地储存，也被称作是 ConnectorMetadata。这是一个包含了诸如 logo，type 等固定参数的对象。（🧐 对上述这些参数有疑问？不要担心，接下来会有详尽的解释！）
  • 远端储存 是一些存储在远端数据库的需要频繁更改的参数。

连接器的 本地储存：ConnectorMetadata​

id​

id 是一个用来在 Logto 中 唯一 标识连接器的字符串类型参数。

他由连接器的开发者指定，并会被同步到数据库中。

target​

target 是用来标识连接器服务提供商的小写英文字符串。

举个例子，如果你使用 Google 账号登录 Logto，则对应的 target 应为「google」。 target 的值可以是任意非空字符串，但由于创建之后不允许修改，我们推荐在设置 target 值时尽量直观。我们不允许同平台下有多个相同的 target 值。想象一下，终端用户企图登录，但发现界面上有两个 GitHub 登录的按钮，这会让用户非常疑惑并且也并不是一个有效的场景。另一方面，你可以有多个不同平台下的社交连接器，他们共享同一个 target。根据「微信」的开发规范，用户在手机上使用微信登录时，我们后台需要一个「原生」微信连接器；而如果用户需要用微信登录网页应用则需要在 Logto 后台创建一个「网页」微信连接器，他们的 target 都是「wechat」。

type​

type 是用以记载连接器类型的属性。

type 的值应该是下列之一：

  • Social：在用户授权的前提下可以访问到任意第三方社交媒体的连接器。
  • SMS：让用户可以在手机上收到短信的连接器。
  • Email：帮助向终端用户发送邮件的连接器。

platform​

platform 记录了连接器使用的平台场景。

platform 的值只能是 null 或者下列的字符之一：

  • Native：只能 协同移动端原生应用的连接器。
  • Web：只能 在桌面端的网页应用使用的连接器。
  • Universal：可以同时适用于桌面端网页应用和移动端网页的连接器。
备注

邮件连接器 和 短信连接器 的 platform 总应该是 null。
只有 社交连接器 可以有非 null 的 platform 值。

name​

name 是一个 key 为 i18n 国家地区代码，value 是连接器用以显示的名称的对象。

description​

description 是一个 key 为 i18n 国家地区代码，value 为连接器简介的对象。

备注

为了更好的支持客户端本地化的展示，我们把 name 和 description 都存成了映射，其中 key 为想要支持的国家地区代码，相对应的 value 是以该国家地区文字形式呈现的连接器名字或简述。

logo​

logo 是一个用以记录连接器的 logo 的相对路径或者 URL 的参数。

logoDark​

logoDark 是一个记录连接器深色模式 logo 的相对路径或者 URL 的参数，可以为空，意味着没有为深色模式单独设计 logo。

备注

logo 总是必填的，同时 logoDark 是选填的。

logo 在浅色模式时显示，而如果在深色模式时 logoDark 存在则会显示，否则会在深色模式显示 logo。

readme​

readme 是一个记录连接器 README 配置说明文档的相对路径。该文档会展示在「管理控制台」内，以引导用户完成必要的连接器配置。

configTemplate​

configTemplate 是一个记录连接器配置样例文件的相对路径，在添加或切换到新的连接器时，会预填在文本输入框中，向用户呈现连接器配置文件的形式。

连接器的 远端储存：数据库​

id​

id 是一个在数据库中可以用以 唯一 确定连接器的参数，并且也是映射 ConnectorMetadata 和连接器数据库参数的桥梁。

enabled​

enabled 是一个默认值为 false 的布尔变量。它是控制连接器使用状态的开关。

通过把该值从 true 置为 false 可以把已配置好正在使用的连接器关闭。

config​

config 可以是一个任意非空的对象。

它是连接器用以储存配置的字段。每个连接器都有不同的 config 参数，并且为了保证连接器能正常使用，在同步至数据库之前都会被强制校验（不同的连接器对于能通过校验有不同的标准），只有通过校验才会被更新至数据库，否则会抛错。

连接器的开发者需要为他们的连接器实现一个 config 的校验标准，详见「创建连接器」。

想要一睹 config 真容？查看「配置连接器」或者「管理控制台」里每个连接器的配置界面。

备注

当前版本的 Logto 只支持同时有一个 短信连接器（或 邮件连接器）的 enabled 为 true，也就意味着一旦一个其他的 短信连接器（或者 邮件连接器）被启用时，其他所有的 短信连接器（或者 邮件连接器）会被关闭（即 enabled 被置为 false）。

上述的 同时只有一个 短信连接器（或者 邮件连接器）能被启用 的约束，对于 社交连接器 不适用。
也就是说，你可以添加并启用多个 社交连接器。

createdAt​

createdAt 是一个自动生成的用以记录连接器创建时间的时间戳。
