🧱 核心服务

简介​

核心服务 是担任 Logto 关键职责的单体服务。源代码位于 /packages/core。

备注

核心服务 与 SDK 核心 是两个概念。阅读 SDK Convention 以了解不同之处。

简化起见，我们将「核心服务」分为四个主要模块：

名称描述载入路径
OIDC Provider一个 OpenID Provider。/oidc
管理 API用于管理 Logto 的 API 集合。要求具有 admin 角色。/api
前端代理用于前端项目的 HTTP 代理或静态文件分发。阅读 前端代理 以了解更多。

包括 OIDC 在内的后端 API 已集成于 core 软件包内；前端代理则依赖于 Logto monorepo 中相关的兄弟软件包。

OIDC Provider​

Logto 在底层使用了超赞的、并经过 OpenID Connect 认证的项目 node-oidc-provider。该 provider 被加载于路径 /oidc；你可以在 packages/core/src/oidc 找到相关的文件和配置。

已启用的 OpenID 功能​

  • OpenID Connect Core
  • OpenID Connect Discovery
  • OpenID Connect RP-Initiated Logout
  • OAuth 2.0 Token Revocation
  • OAuth 2.0 Resource Indicators

管理 API​

管理 API 是一个用于管理和更新 Logto 数据的 API 集合。只有具有 admin 角色的用户可以访问。

它被加载于路径 /api。前往 API 参考 以在浏览器中了解细节，或者通过 GET /api/swagger.json 来获取 JSON 格式的 Swagger 数据。

认证（Authentication）​

「管理控制台」非常依赖这些 API，当然你也可以手动调用。

你需要一个满足如下条件的 Access Token：

  • 是一个有效的 token。
  • 颁发自你的 Logto 实例。
  • 它的 audience（aud，即 API indentifier）指向 https://api.logto.io。
  • 它的 role_names 数组里包含 "admin"。

将 token 以 Bearer 格式（Bearer YOUR_TOKEN）置于 HTTP headers 里的 Authorization 字段中即可。

用户 API​

我们提供标准的 OIDC Userinfo Endpoint，被加载于路径 /oidc/me.

认证（Authentication）​

你需要一个满足如下条件的 Access Token：

  • 是一个有效的 token。
  • 颁发自你的 Logto 实例。
  • 它的 audience（aud，即 API indentifier）指向 https://api.logto.io。

将 token 以 Bearer 格式（Bearer YOUR_TOKEN）置于 HTTP headers 里的 Authorization 字段中即可。

前端代理​

前端代理 是一个用于分发前端项目的中间件函数，根据环境执行：

  • 如果是开发环境，它将转发 HTTP 请求至前端开发服务器。
  • 如果是生产环境，它将直接分发静态前端文件。

Logto 含有三个前端代理：

名称前端软件包加载路径
「管理控制台」代理/packages/console/console
演示应用代理/packages/demo-app/demo-app
UI（登录体验）代理/packages/ui/
备注

你也许注意到了 UI 代理使用了根路径。与其他代理不同，UI 代理是一个备选（fallback）代理，也就是说只有当其他代理都不匹配时，该代理才会生效。
